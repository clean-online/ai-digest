Anthropic Engineering Official Blog

An update on recent Claude Code quality reports — A detailed postmortem of three separate issues that affected Claude Code, the Claude Agent SDK, and Claude Cowork in March-April 2026. The issues: (1) Default reasoning effort changed from high to medium, hurting latency but reducing quality — reverted after user feedback. (2) A bug in session context clearing that made Claude seem forgetful — fixed April 10. (3) A system prompt change to reduce verbosity that hurt coding quality — reverted April 20. Key takeaway: the aggregate effect of small, well-intentioned changes created the perception of broad degradation. Anthropic now has better monitoring to distinguish signal from normal variation.

An update on recent Claude Code quality reports — 一篇详细的事后分析报告，涉及 2026 年 3 月至 4 月影响 Claude Code、Claude Agent SDK 和 Claude Cowork 的三个独立问题。问题包括：（1）默认推理强度从高能级改为中能级，减少了延迟但降低了质量——根据用户反馈已回滚。（2）会话上下文清除中的一个 bug 让 Claude 看起来健忘——4 月 10 日修复。（3）为减少冗长而做的系统提示更改损害了编码质量——4 月 20 日回滚。关键要点：微小、善意的更改的累积效应造成了广泛降级的感知。Anthropic 现在有了更好的监控来区分信号和正常变异。

Anthropic Engineering Official Blog

Scaling Managed Agents: Decoupling the brain from the hands — A technical deep dive into Claude Managed Agents, a hosted service that runs long-horizon agents. The core insight: the authors previously found that Claude Sonnet 4.5 would wrap up tasks prematurely due to "context anxiety" — and addressed it with context resets in the harness. But with Claude Opus 4.5, that behavior was gone — the resets became dead weight. This illustrates the key design principle: Managed Agents virtualize agent components (session, harness, sandbox) so implementations can be swapped without disturbing others. Like operating system abstractions (process, file) that outlasted hardware, these abstractions outlast model versions.

Scaling Managed Agents: Decoupling the brain from the hands — 一篇关于 Claude Managed Agents 的技术深度文章，这是一项运行长周期 agent 的托管服务。核心洞察：作者之前发现 Claude Sonnet 4.5 会因为"上下文焦虑"而过早结束任务——并通过在 harness 中添加上下文重置来解决这个问题。但到了 Claude Opus 4.5，这种行为消失了——重置变成了死重。这说明了关键设计原则：Managed Agents 将 agent 组件（会话、harness、沙箱）虚拟化，以便可以在不干扰其他组件的情况下交换实现。就像超越硬件的操作系统抽象（进程、文件）一样，这些抽象超越了模型版本。

Claude Blog Official Blog

New in Claude Managed Agents: self-hosted sandboxes and MCP tunnels — A major update to Claude Managed Agents: starting today, agents can execute in a sandbox you control and connect to your private MCP servers. Two key features: (1) Self-hosted sandboxes — the agent loop stays on Anthropic's infrastructure, but tool execution moves to your environment. This means sensitive files, packages, and services stay within your perimeter. Supports Cloudflare, Daytona, Modal, or Vercel as sandbox providers. (2) MCP tunnels — agents can connect to your private Model Context Protocol servers, accessing internal services securely. This is a significant step toward enterprise-grade agent deployment with proper security and compliance controls.

New in Claude Managed Agents: self-hosted sandboxes and MCP tunnels — Claude Managed Agents 的一次重大更新：从今天开始，agent 可以在你控制的沙箱中执行，并连接到你的私有 MCP 服务器。两个关键功能：（1）自托管沙箱——agent 循环留在 Anthropic 的基础设施上，但工具执行移动到你的环境。这意味着敏感文件、包和服务保留在你的边界内。支持 Cloudflare、Daytona、Modal 或 Vercel 作为沙箱提供商。（2）MCP 隧道——agent 可以连接到你的私有 Model Context Protocol 服务器，安全地访问内部服务。这是朝着具有适当安全和合规控制的企业级 agent 部署迈出的重要一步。
