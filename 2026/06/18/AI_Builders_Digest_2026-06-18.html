Simulating Humans at Scale: Simile's Joon Sung Park
TAKEAWAY Joon Sung Park, founder of Simile, argues that we're finally ready to build the "simulation pillar" of sci-fi societies — LLMs now encode enough human behavior to create generative agents that form believable micro-behaviors, and Smallville was just the beginning.

From Smallville to Simile. Park's Stanford experiment "Smallville" (Apr 2023) proved that LLMs can encode human behavior from web data and generate believable micro-behaviors — what would person X do in situation Y? That insight became the foundation for Simile, an applied AI lab simulating human behavior and societies at scale.

Why simulations matter. Sci-fi societies always have two pillars: AGI and simulations that guide society. We now have the first real crack at building the simulation pillar. The conviction: simulations won't just be games — they'll guide real-world decisions by modeling how humans actually behave, not how they say they behave.

The current moment. Five years ago this would've been impossible. Today, the combination of LLMs + agentic frameworks makes it feasible to simulate not just individuals but societies — groups, cultures, emergent behaviors. Simile is building the infrastructure layer for this: applied simulation as a service.

从 Smallville 到 Simile。 Park 在斯坦福的"Smallville"实验（2023 年 4 月）证明了 LLM 可以从网络数据中编码人类行为，并生成可信的微行为——情况 X 下，人物 Y 会做什么？这一洞察成为了 Simile 的基础，这是一家大规模模拟人类行为和社会的应用 AI 实验室。

为什么模拟很重要。 科幻社会总是有两个支柱：AGI 和引导社会的模拟。我们现在第一次真正有机会构建模拟支柱。核心信念：模拟不仅仅是游戏——它们将通过建模人类的实际行为（而非他们声称的行为）来引导现实世界的决策。

当前时刻。 五年前这还是不可能的事。今天，LLM + agentic 框架的结合使得不仅模拟个人，还能模拟社会——群体、文化、涌现行为——成为可能。Simile 正在为此构建基础设施层：应用模拟即服务。

Claude Blog · Published Jun 08, 2026
Building intelligent apps for Apple platforms with Claude in the Foundation Models framework

Claude meets Apple's on-device models. Anthropic released Foundation Models framework support for Claude via a new Swift package. Apple developers can now use Apple's Foundation Models framework (iOS 26+, etc.) to call Claude for complex workflows — multi-step reasoning, code generation, web search, and data analysis — while keeping fast local tasks on-device.

How it works. Apple's framework returns typed Swift values from @Generable annotations, so developers arrive at the Claude API call with clean inputs instead of raw user text. Claude's response streams back into the same view. One experience for the user, backed by the right model for each step: on-device for speed, Claude for depth.

What this unlocks. A journaling app can generate daily prompts on-device, then ask Claude to find threads across months of entries. A study app can define a term on-device, then hand off to Claude when the student asks "why does this matter for everything else we've covered?" It's a hybrid architecture that plays to each model's strengths.

Claude 接入 Apple 设备端模型。 Anthropic 通过新的 Swift 包发布了 Claude 对 Foundation Models 框架的支持。Apple 开发者现在可以使用 Apple 的 Foundation Models 框架（iOS 26+ 等）调用 Claude 处理复杂工作流——多步推理、代码生成、网络搜索和数据分析——同时将快速本地任务保留在设备上。

工作原理。 Apple 的框架通过 @Generable 注解返回类型化的 Swift 值，因此开发者在调用 Claude API 时拥有清晰的输入，而非原始用户文本。Claude 的响应流式传回同一视图。对用户来说是一个体验，每个步骤都由合适的模型支撑：设备上处理速度，Claude 处理深度。

这意味着什么。 日记应用可以在设备上生成每日提示，然后让 Claude 在数月的条目中寻找线索。学习应用可以在设备上定义术语，然后当学生问"为什么这对我们涵盖的其他所有内容都很重要？"时交给 Claude。这是一种混合架构，发挥了每个模型的优势。
