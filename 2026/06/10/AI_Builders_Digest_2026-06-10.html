No Priors

Building an AI Guardian for Enterprise with Onyx Security CEO Maxim Bar Kogan

The Takeaway As enterprises deploy AI agents at exponential scale, the attack surface is growing faster than anyone anticipated. Onyx Security is building "guardian agents" that watch the agents — using historical behavioral data to detect and prevent illegitimate or incorrect agent actions before they cause damage. The core insight: enterprises won't let Anthropic or OpenAI see their historical agent data (too data-hungry), so the security layer must be independent.

Maxim Bar Kogan, co-founder and CEO of Onyx Security, joins No Priors to talk about building agents that watch the AI agents. The conversation starts with a framing that should make every CTO nervous: as you exponentially do more things with AI, you're going to start having really bad actions happen. Enterprises are realizing that risk has grown exponentially and they don't have any way to stop adoption — they just now have to do something to reduce the chance of illegitimate or incorrect agent actions.

The core technical insight: Onyx uses a lot of historical data on how agents have behaved to detect anomalies. But here's the problem — enterprises today are not willing to let Anthropic or OpenAI have that historical data, because they know these are very data-hungry companies that will want to train on that data. This creates a massive opening for an independent security layer that watchers the agents without consuming the data for model training.

The conversation also touches on specialized model training, mythos, alignment research, and the Israeli ecosystem in security and now AI. Maxim's background spans the Israeli defense and security world, and Onyx is staffed with researchers, mathematicians, and engineers who understand both adversarial AI and enterprise security. The episode is a compelling look at the next layer of AI infrastructure: not building bigger models, but building the guardrails that let enterprises actually deploy them.

Onyx Security 联合创始人兼 CEO Maxim Bar Kogan 参加 No Priors，谈论构建监视 AI agent 的 agent。对话从一个应该让每个 CTO 都紧张的框架开始：当你用 AI 指数级地做更多事情时，你将会开始遇到真正糟糕的行为。企业正在意识到风险已经指数级增长，但他们没有任何方式阻止采用——他们现在只是必须做些什么来减少 agent 行为不合法或不正确的可能性。

核心技术洞察：Onyx 使用大量关于 agent 如何行事的历史数据来检测异常。但这里有个问题——当今的企业不愿意让 Anthropic 或 OpenAI 拥有这些历史数据，因为他们知道这些是非常渴求数据的公司，会想要用那些数据训练。这为独立的、监视 agent 但不消耗数据用于模型训练的安全层创造了巨大的开放空间。

对话还涉及专业模型训练、神话、对齐研究，以及以色列在安全和现在 AI 方面的生态系统。Maxim 的背景涵盖以色列国防和安全世界，Onyx 配备了理解对抗性 AI 和企业安全的研究者、数学家和工程师。这期节目令人信服地审视了 AI 基础设施的下一层：不是构建更大的模型，而是构建让企业实际部署它们的护栏。

Claude Blog

Building intelligent apps for Apple platforms with Claude in the Foundation Models framework

Anthropic is releasing Foundation Models framework support for Claude through a new Swift package that lets Apple developers use Apple's Foundation Models framework to call Claude for more complex workflows. Apple's Foundation Models framework gives developers access to tap into models natively from Swift — it is very easy to use and can return typed Swift values through guided generation in as few as three lines of code.

Developers can use this to tap into Apple's on-device models for fast, local tasks like summarization or extraction. Then, when a request calls for multi-step reasoning, code generation, or more, developers can now hand off to Claude — seamlessly, within the same Swift workflow. Because Apple's framework returns typed Swift values from @Generable annotations, developers arrive at the Claude API call with clean inputs instead of raw user text.

What this unlocks: The Foundation Models framework already powers a range of intelligent on-device features — journaling apps that surface personalized prompts, document apps that summarize contracts, learning apps that explain a concept at a student's level. Adding Claude extends each of those patterns. A journaling app can generate daily prompts on-device, then ask Claude to find threads across months of entries. A study app can define a term on-device, then hand off to Claude when the student follows up with "why does this matter for everything else we've covered?" It's one experience for the user, backed by the right model for each step.

The feature will be available tomorrow and works through Apple's Foundation Models framework on iOS 27, iPadOS 27, macOS 27, visionOS 27, and watchOS 27. Add it to your project, sign in with an Anthropic API key, and pass typed Swift values to Claude.

Anthropic 通过一个新的 Swift 包发布了对 Claude 的 Foundation Models 框架支持，让 Apple 开发者使用 Apple 的 Foundation Models 框架调用 Claude 处理更复杂的工作流。Apple 的 Foundation Models 框架让开发者能够从 Swift 原生访问模型——它非常容易使用，可以通过引导生成以少至三行代码返回类型化的 Swift 值。

开发者可以用这个来利用 Apple 的设备上模型处理快速的本地任务，比如总结或提取。然后，当一个请求需要多步推理、代码生成或更多时，开发者现在可以无缝地在同一 Swift 工作流中将任务交给 Claude。因为 Apple 的框架从 @Generable 注释返回类型化的 Swift 值，开发者带着干净的输入到达 Claude API 调用，而不是原始用户文本。

这解锁了什么：Foundation Models 框架已经为数智能设备上功能提供动力——显示个性化提示的日志应用、总结合同的文档应用、以学生水平解释概念的学习应用。添加 Claude 扩展了这些模式中的每一个。日志应用可以在设备上生成每日提示，然后请 Claude 在数月的条目中找到线索。学习应用可以在设备上定义一个术语，然后当学生跟进"为什么这对我们涵盖的其他一切都重要？"时交给 Claude。对用户来说这是一个体验，每个步骤都由正确的模型支持。

该功能将于明天推出，并通过 Apple 的 Foundation Models 框架在 iOS 27、iPadOS 27、macOS 27、visionOS 27 和 watchOS 27 上工作。将其添加到你的项目中，使用 Anthropic API 密钥登录，然后将类型化的 Swift 值传递给 Claude。
