Andrej Karpathy AI Researcher / Deep Learning

Karpathy shares a practical LLM workflow: when the model needs more context than you're willing to type, switch to voice and just ramble for ~10 minutes — a messy, free-association stream. He finds LLMs are surprisingly good at reconstructing coherent intent from long, incoherent rambles, sometimes turning it into a short back-and-forth interview.

Karpathy 分享了一个实用的 LLM 工作流：当模型需要比你想打出来的更多上下文时，切换到语音、随口嘟囔约 10 分钟——混乱、自由联想的意识流。他发现 LLM 出奇地擅长从冗长、 incoherent 的嘟囔中重建出连贯意图，有时还能把它变成几轮简短的问答式访谈。

Factory's Matan Grinberg: The Coming ‘Dark Factory’ Where Software Builds Itself

The Takeaway Factory CEO Matan Grinberg argues software development is heading toward a "dark factory" — lights off, droids autonomously building software. Enterprises want model-agnostic agents to avoid vendor lock-in; Factory's router dynamically picks models per task. He predicts 90% of tokens will be asynchronous (autonomous) within 12–24 months.

Training Data sits down with Matan Grinberg, cofounder & CEO of Factory, which builds "droids" — autonomous agents for software development. Grinberg frames the journey as "our journey in the desert": they started in April 2023 betting on fully autonomous agents two years before enterprises or developers were ready. The interaction pattern was too ambitious too early; the breakthrough came on 09/26/2025 with the droid CLI, which met developers where they were and was model-agnostic and state-of-the-art.

The biggest shift wasn't the models — it was developer behavior. Enterprises, burned by single points of failure, refuse to put their fate in any one model lab. Factory's answer is the Factory router: dynamically route to different models per task (Gemini Flash for vibe-coded dashboards, a fine-tuned model for COBOL, generate with OpenAI / test with Anthropic / review with Gemini). Routing instructions can even be given in natural language.

On the economics: the question for CIOs is shifting from "where do we put every incremental token?" to "where do we put every incremental dollar?" — headcount vs tokens, per part of the org. Engineering teams scale with tokens; sales teams scale with humans. You can only answer this with feedback loops that close the loop on whether software actually delivered business outcome.

The name "Factory" borrows from Tesla: a dark factory where the lights are off and robotic arms just build. Grinberg predicts that within 12–24 months, ~90% of tokens will be asynchronous — droids autonomously finding signal and shipping first-pass solutions, not humans kicking off each task. That's where agent-native software development truly begins.

Training Data 本期嘉宾是 Factory 联合创始人兼 CEO Matan Grinberg，Factory 做的是「droids」——用于软件开发的自主 agent。Grinberg 把这段历程称为「沙漠之旅」：他们 2023 年 4 月就押注全自主 agent，比企业或开发者准备好早了两年。早期交互模式过于激进；转机出现在 2025-09-26 的 droid CLI——在开发者当前位置接住他们，且模型无关、达到 SOTA。

最大的变化不是模型，而是开发者行为。企业被单点故障坑过，拒绝把命运押在任何一个模型实验室身上。Factory 的解法是 Factory router：按任务动态路由到不同模型（vibe coding 看板用 Gemini Flash，COBOL 用微调模型，OpenAI 生成 / Anthropic 测试 / Gemini 审查）。路由指令甚至可以用自然语言描述。

关于经济学：CIO 的问题正从「每多一个 token 投在哪？」转向「每多一美元投在哪？」——人头 vs token，按组织板块分配。工程团队随 token 扩张；销售团队随人力扩张。只有当反馈闭环能验证软件是否真的交付了业务结果，才能回答这个问题。

「Factory」这个名字借自特斯拉：暗工厂里灯关着，机械臂自己造东西。Grinberg 预测 12–24 个月内约 90% 的 token 将是异步的——droids 自主发现信号、产出初版方案，而不是人类逐项启动任务。那才是 agent-native 软件开发的真正起点。

Building intelligent apps for Apple platforms with Claude in the Foundation Models framework

Anthropic released Foundation Models framework support for Claude via a new Swift package. Apple developers can use Apple's on-device Foundation Models framework to call Claude for multi-step reasoning, code generation, and more. Because the framework returns typed Swift values from @Generable annotations, developers arrive at the Claude API call with clean structured inputs instead of raw text.

On-device models handle fast local tasks (summarization, extraction); Claude is handed off when a request needs deeper reasoning. Claude can also search the web and execute code for data analysis, streaming responses back into the same SwiftUI view. The package works on iOS 27 / iPadOS 27 / macOS 27 / visionOS 27 / watchOS 27.

Anthropic 通过一个新的 Swift 包发布了 Claude 对 Foundation Models 框架的支持。Apple 开发者可用设备端 Foundation Models 框架调用 Claude，处理多步推理、代码生成等。由于框架从 @Generable 注解返回类型化的 Swift 值，开发者带着干净的结构化输入（而非原始文本）进入 Claude API 调用。

设备端模型负责快速的本地任务（摘要、抽取）；当请求需要更深推理时交给 Claude。Claude 还能联网搜索、执行代码做数据分析，并把响应流式回同一个 SwiftUI 视图。该包支持 iOS 27 / iPadOS 27 / macOS 27 / visionOS 27 / watchOS 27。
