AI & I by Every

Inside Stainless: The Developer Tools Startup Anthropic Just Bought for $300 Million

The Takeaway The $300M Stainless acquisition isn't about a product — it's about owning the plumbing layer of the AI-native internet. Whoever controls how APIs and MCP servers are built decides how every AI agent talks to every service. Anthropic just bought themselves a seat at that table, and they got it from under OpenAI and Google, who were Stainless' biggest customers.

Alex Rattray built Stainless into the behind-the-scenes infrastructure for the AI era: they write the APIs and SDKs that OpenAI, Anthropic, and every major AI company use to expose their services. Before Stainless, Rattray worked at Stripe building their API — the attention to developer experience carried over. The conversation with Dan Shipper was recorded before the acquisition, but reads like a blueprint for what Anthropic just bought.

The core technical insight: MCP servers are deceptively hard to build well. The temptation is to expose every API endpoint as a tool, but that overwhelms the model. Rattray's approach: keep toolsets small, give each tool a precise name and description, minimize inputs/outputs, and add a JSON filter to strip unnecessary data. For complex APIs, Stainless uses "dynamic mode" — three tools only: list endpoints, pick one and learn about it, execute it.

On MCP as business infrastructure: Rattray uses MCP servers to connect Notion and HubSpot, creating a "business copilot" where queries like "which customers signed up last week?" cross multiple databases and return a synthesized answer. More ambitiously, he built a shared "company brain" using Claude Code — saving useful inputs (customer feedback, SQL queries) into GitHub, creating a curated archive the whole team can query.

The forward-looking take: MCP's future is code execution, not tool-calling. Instead of giving models hundreds of tools, the direction is letting them write and execute code that calls APIs — more flexible, more auditable, and more scalable. The current MCP spec has real limitations around security, state management, and discoverability that the ecosystem is still working through.

Alex Rattray 把 Stainless 做成了 AI 时代的幕后基础设施：他们为 OpenAI、Anthropic 和所有主流 AI 公司编写 API 和 SDK。在创立 Stainless 之前，Rattray 在 Stripe 做 API——对开发者体验的执念延续了下来。这段与 Dan Shipper 的对话录制于收购之前，但读起来就像 Anthropic 刚买下的那张蓝图。

核心技术洞察：MCP Server 做好非常困难。诱惑是把每个 API 端点都暴露为工具，但那样会把模型淹没。Rattray 的方法：保持工具集精简，给每个工具精确的名称和描述，最小化输入/输出，加 JSON 过滤器去掉冗余数据。对于复杂 API，Stainless 用"动态模式"——只有三个工具：列出端点、选一个了解它、执行它。

关于 MCP 作为商业基础设施：Rattray 用 MCP Server 连接 Notion 和 HubSpot，创造了一个"商业副驾"，像"上周哪些客户注册了？"这样的查询能跨数据库返回综合答案。更有野心的是，他用 Claude Code 构建了一个共享的"公司大脑"——把有用的输入（客户反馈、SQL 查询）保存到 GitHub，形成团队可查询的精选知识库。

前瞻判断：MCP 的未来是代码执行，而非工具调用。方向不是给模型成百上千个工具，而是让它们写代码来调用 API——更灵活、更可审计、更可扩展。当前 MCP 规范在安全、状态管理和可发现性方面还有真实限制，生态系统仍在摸索中。
