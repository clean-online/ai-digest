Aaron Levie Box CEO

Levie identifies the core bottleneck for enterprise AI agent deployment: context quality. Too much information or conflicting sources → agents produce wrong results. Levie 指出了企业 AI Agent 部署的核心瓶颈：上下文质量。信息过多或来源冲突 → Agent 产出错误结果。信息过少 → Agent 无法交付价值。真正的洞察：大多数 AI 战略失败实际上是数据战略失败的伪装。为 Agent 正确配置结构化和非结构化数据环境，现在是企业采用 AI 时杠杆最高的工作。

Anthropic Engineering Official Blog

Anthropic published a detailed postmortem of recent Claude Code quality reports. Three separate issues were identified and all resolved as of April 20 (v2.1.116): (1) default reasoning effort changed from high to medium on March 4, reverted April 7 after users reported preference for higher intelligence; (2) a bug in idle session memory clearing caused repetitive behavior, fixed April 10; (3) a system prompt change to reduce verbosity (April 16) hurt coding quality, reverted April 20. Key lesson: each change affected different traffic slices, making aggregate degradation hard to diagnose.

Anthropic 发布了一篇详细的 Claude Code 质量报告事后分析。识别出了三个独立问题，截至 4 月 20 日（v2.1.116）已全部修复：(1) 3 月 4 日默认推理努力从高改为中，4 月 7 日用户反馈更倾向于高智能后回滚；(2) 空闲会话记忆清除的 bug 导致重复行为，4 月 10 日修复；(3) 4 月 16 日减少冗余度的系统提示变更损害了编码质量，4 月 20 日回滚。关键教训：每次变更影响了不同的流量切片，使得聚合退化难以诊断。

Claude Blog Anthropic 官方博客

Built-in memory for Claude Managed Agents is now in public beta. Memories are stored as files on a filesystem, giving developers full control: export, manage via API, audit logs, roll back versions. Multiple agents can work concurrently against the same memory store with different access scopes (e.g. org-wide read-only, per-user read-write). The system is optimized against internal benchmarks for long-running agents that improve across sessions. This is enterprise-grade memory infrastructure, not a toy.

Claude Managed Agents 的内置记忆现在进入公开 Beta。记忆以文件形式存储在文件系统上，给予开发者完全控制权：导出、通过 API 管理、审计日志、回滚版本。多个 Agent 可以针对不同访问权限（如全组织只读、每用户读写）并发操作同一记忆存储。系统针对跨会话改进的长时间运行 Agent 进行了内部基准优化。这是企业级记忆基础设施，不是玩具。

AI & I by Every

The Secrets of Claude's Platform From the Team Who Built It

The Takeaway The next platform leap is Claude becoming self-aware enough to write itself — the model understands its own architecture well enough to spin up the right sub-agents, pick the right models, and dynamically assemble the architecture you need on the fly.

Angela (Head of Product for Claude Platform) and Caitlin (Head of Engineering for Claude Platform) at Anthropic sat down with the AI & I show to talk about where the Claude platform is heading. The conversation starts with a simple framing: a year from now, the platform will be where Claude understands itself so well it can write itself on the fly.

The evolution narrative is worth tracing: in the GPT-3 days, the platform was a completion endpoint — send a prompt, get a response. After that: completion endpoints with tool calling, chat sessions. Now: Claude Managed Agents, which is essentially a cloud computer with memory and persistent context. The platform scaled from a single API call to a full runtime environment.

The deepest insight: the platform's next phase isn't about more models or bigger context — it's about the model understanding its own capabilities. When Claude knows what architectures exist, what models to route to, and how to orchestrate sub-agents, the "platform" becomes an intelligent orchestrator rather than a static API. This is why the team is so focused on Claude's ability to "understand itself."

On the scaling challenge: as agents become more autonomous and the platform runs more of them concurrently, the infrastructure requirements are non-trivial. Caitlin's engineering team is thinking about how to make the platform scale to orders-of-magnitude more agents without degrading response quality or increasing latency.

Anthropic 的 Claude 平台产品负责人 Angela 和工程负责人 Caitlin 做客 AI & I 节目，畅谈 Claude 平台的未来方向。对话从一个简单的框架开始：一年后，平台将发展到 Claude 对自己的理解足够好，可以即时自我编写的程度。

演进叙事值得追溯：在 GPT-3 时代，平台是一个补全端点——发送提示词，获取回复。之后：带工具调用、聊天会话的补全端点。现在：Claude Managed Agents，本质上是一台带记忆和持久上下文的云计算机。平台从单次 API 调用扩展到了完整的运行时环境。

最深刻的洞察：平台的下一个阶段不在于更多模型或更大上下文——而在于模型理解自身能力。当 Claude 知道存在哪些架构、应该路由到哪些模型、如何编排子 Agent 时，"平台"就变成了智能编排器，而非静态 API。这就是为什么团队如此专注于 Claude"理解自身"的能力。

关于扩展挑战：随着 Agent 变得更加自主，平台并发运行更多 Agent，基础设施要求不容小觑。Caitlin 的工程团队正在思考如何让平台扩展到数量级更多的 Agent，同时不降低回复质量或增加延迟。
